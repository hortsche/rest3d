/0/search:

Note: the server simulate a search by doing a http query to Google Warehouse, 3dvia, and ourbricks. Then the server interprets the html pages and format into a {xml,json,html} result. Therefore this prototype implementation is way slower than it would be if the servers would have rest3D direct access.

  • Search string:
  • 0/search.xml?q=...
  • 0/search.html?q=...
  • 0/search.json?q=...
